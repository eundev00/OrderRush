## 1. 프로젝트 개요

모바일 터치 기반 입력과 행동 큐 시스템을 활용한 주방 시뮬레이션 게임. 레퍼런스: PlateUp!

## 2. 스토리텔링

### 배경

요리학교를 갓 졸업한 신인 셰프가 전 재산을 털어 낡은 동네 식당을 인수한다. 설레는 첫날부터 임대료 고지서가 날아오는 날까지, 15일의 영업이 시작된다. 매일 손님을 맞이하고, 번 돈으로 식당을 조금씩 키워나가고, Day 15가 끝나면 임대료를 낸다.

### 핵심 동기

매일 영업 → 코인 획득 → 상점에서 식당 업그레이드 → 더 많은 손님 → 임대료 납부

### Day별 스토리 흐름

| 구간      | 스토리                                     | 분위기     |
| --------- | ------------------------------------------ | ---------- |
| Day 1~3   | 첫 오픈. 손님이 적지만 설레는 날들         | 기대감     |
| Day 4~6   | 단골이 생기기 시작. 입소문이 퍼진다        | 성장       |
| Day 7~10  | 맛집 블로거 방문. 갑자기 손님 폭발         | 긴장감     |
| Day 11~13 | 임대료 고지서 도착. 이제 진짜 승부         | 위기       |
| Day 14~15 | 마지막 영업일. 단골 손님들이 모두 몰려온다 | 클라이맥스 |

### 단골 캐릭터

| 캐릭터    | patience 비율 | 특징                         | 패턴          |
| --------- | ------------- | ---------------------------- | ------------- |
| 할머니    | 1.5x          | patience 넉넉, 항상 스테이크 | 안정형        |
| 직장인    | 0.7x          | patience 짧음, 성공 시 팁 多 | 고위험 고보상 |
| 꼬마      | 1.0x          | 메뉴 랜덤                    | 변수형        |
| 일반 손님 | 1.0x          | 기본                         | —             |

### Day별 이벤트

- 스토리 흐름에 맞춰 구간별 이벤트 발생
- 이벤트 상세 내용 추후 확정
  
### 연출 방향

- Day 시작 시 로딩 화면 한 줄 텍스트로 스토리 전달
    - 예) "Day 7 — 오늘은 블로거가 온다는 소문이 있다."
- Day 클리어 시 짧은 메시지 팝업
    - 예) "오늘도 버텼다. 내일은 좀 더 나아질 거야."
- Day 15 클리어 엔딩
    - "임대료를 냈다. 식당은 계속된다."

## 3. 게임 루프

### 전체 구조

| 단계   | 내용                                                                                   |
| ------ | -------------------------------------------------------------------------------------- |
| ① 상점 | Day 시작 전 카드 구매 (첫날 제외)                                                      |
| ② 영업 | 타임바 진행 중 손님 균등 스폰 → 타임바 종료 후 입장 마감 → 남은 손님 처리 시 하루 종료 |
| ③ 정산 | 코인 획득 + 스토리 메시지 → 다음 Day                                                   |

### 타임바 & 최대 손님 수

- 타임바: Day 1 기준 100초, 3일마다 25초 증가
- 최대 손님 수: Day 1 기준 4명, 3일마다 1명 증가

### 실패 조건

- 손님 patience 게이지 소진 → 해당 Day 처음부터 재시작

### 런 종료 조건

| 상황                        | 결과                             |
| --------------------------- | -------------------------------- |
| Day 15 클리어 + 임대료 충분 | 엔딩 → 다음 런 시작              |
| Day 15 클리어 + 임대료 부족 | 다음 런 시작, 보유 카드 1장 몰수 |
| 영업 중 patience 소진       | 해당 Day 처음부터 재시작         |

### 런 간 이월

- 코인 이월 O
- 임대료 런마다 상승

## 4. 손님 시스템

### patience 구조

- 단계별 기본 시간 × 캐릭터 비율 = 실제 patience 시간
- 그룹 내 여러 캐릭터가 있을 경우 가장 낮은 비율 적용
- 단계: 입장 대기 / 주문 대기 / 음식 대기 각각 별도 관리
- 음식 대기 중 그룹원 1명 서빙 완료 시마다 patience 게이지 소량 회복
- 그룹 전원 서빙 완료 시 식사 단계로 전환

### 코인 지급

- 손님 퇴장 시 메뉴 코인 지급
- 직장인 서빙 성공 시 메뉴 코인의 10% 추가 팁 지급

### 그룹 사이즈 규칙

- 최대 그룹 사이즈 = 배치된 테이블 중 가장 큰 좌석 수
- 2인 테이블만 있으면 최대 2인 그룹
- 4인 테이블 추가 시 최대 4인 그룹 등장

## 5. 메뉴

| 메뉴            | 조리 난이도                 | 코인 |
| --------------- | --------------------------- | ---- |
| 스테이크        | 낮음 (굽기만)               | 100  |
| 버섯 스테이크   | 중간 (굽기 + 버섯 볶기)     | 150  |
| 어니언 스테이크 | 중간 (굽기 + 양파 슬라이스) | 150  |

## 6. 상점 시스템

### 기본 규칙

- Day 시작 전 진입 (첫날 제외)
- 3장 카드 중 1장을 코인으로 구매
- 리프레시 가능 — 횟수마다 비용 누적
- 패스 가능

### 리프레시 비용

| 횟수 | 비용     |
| ---- | -------- |
| 1회  | 50 코인  |
| 2회  | 100 코인 |
| 3회  | 150 코인 |

### 카드 종류

| 카드 타입       | 예시                                             | 지속   |
| --------------- | ------------------------------------------------ | ------ |
| 도구 업그레이드 | 조리 시간 단축, 세척 시간 단축 등 시간 효율 향상 | 영구   |
| 메뉴 추가       | 버섯 스테이크, 어니언 스테이크 언락              | 영구   |
| 알바 채용       | 홀 알바, 부엌 알바 등 (확장 예정)                | 당일만 |
| 테이블 추가     | 2인 테이블, 4인 테이블                           | 영구   |
